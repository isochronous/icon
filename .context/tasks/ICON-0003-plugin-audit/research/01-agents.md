## Improvement Opportunities

1. **IO-1 · Collapse common-constraints to a one-line pointer per agent.** The 9× byte-identical 24-line block (~216 lines repo-wide) could collapse to a single `<!-- common-constraints: see shared/common-constraints.md -->` line in each agent, with the actual constraints resident only in the shared file. Closes M-A2 and reclaims ~192 lines of always-loaded duplicate text. Requires loader behavior change (or pre-commit-hook substitution at distribution time) so the constraints are still present in agent context at run-time — the design decision is between "include at author time" (current) and "include at load time" (proposed). **Effort: medium · Impact: high (token economy across every agent invocation).**
2. **IO-2 · Add a balanced-code-fence lint to a pre-commit check.** A 5-line `grep -c '^```'` script on `agents/*.agent.md` would have caught M-A1 at commit time — odd fence counts indicate imbalance. Same script over `skills/*/SKILL.md` would generalize. Closes M-A1 and prevents recurrence. **Effort: trivial · Impact: medium (single class of defect, persistent across audits).**
3. **IO-3 · Soften manager hardcoded "3+ attempts" / "3 failures" to a named threshold or tunable.** Replace the two-site magic number at `:182` and `:247` with "after repeated failures (typically 2-3)" or a single named tier referenced by both. Closes m-A1. **Effort: trivial · Impact: low (single Minor across two lines).**
4. **IO-4 · Reorder PM Session Start to precede `## When to Invoke`.** Closes m-A5; a 4-line move puts the "MANDATORY FIRST ACTION" mandate at the document position that matches its self-claim. **Effort: trivial · Impact: low (cosmetic).**
5. **IO-5 · Add a `## Common Constraints (apply at all times)` step to PM Session Start.** Closes m-A4; restores symmetry with manager. Two-line edit. **Effort: trivial · Impact: low (cosmetic).**
6. **IO-6 · Promote architect AR table to two sub-tables: "Premature Abstraction" and "Premature Scaling".** The 12-row table is functionally split already (rows L107/L112/L115/L116 = abstraction; L109/L111/L113/L114 = scaling-shaped; L105/L106/L108/L110 = miscellaneous). Splitting into named sub-tables makes the over-density a structural feature rather than a defect, and gives reviewers a hook for "is this in the right sub-table?" Closes M-A3 without trimming rows. **Effort: low · Impact: medium (converts a recurring carry-forward into a navigable structure).**
